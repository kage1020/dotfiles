始業時のブリーフィングを行う。直近の日報から昨日の振り返りと今日の予定を出力する。

## 手順

### Phase 1: 日報の収集

1. `~/.claude/reports/` から直近 5 日分の日報ファイルを探索する:

   ```bash
   for i in $(seq 1 5); do
     d=$(date -v-${i}d +%Y-%m-%d 2>/dev/null || date -d "-${i} days" +%Y-%m-%d)
     [ -f "$HOME/.claude/reports/daily-report-${d}.md" ] && echo "$HOME/.claude/reports/daily-report-${d}.md"
   done
   ```

2. 見つかった日報を新しい順に読み込む（最大 3 日分）。

3. 日報が 1 件も見つからない場合は、集約スクリプトで直近のログを取得してフォールバックする:

   ```bash
   python3 ~/.claude/hooks/aggregate-daily-log.py --date "$(date -v-1d +%Y-%m-%d 2>/dev/null || date -d '-1 days' +%Y-%m-%d)"
   ```

### Phase 2: 分析と出力

日報の内容を分析し、以下のフォーマットで出力する。ファイルには保存しない（ターミナルに出力のみ）。

```markdown
## Morning Briefing — {YYYY-MM-DD}

### 昨日の振り返り

**やったこと**:
- 日報の「作業内容」セクションから要点を抽出（1タスク1行、簡潔に）

**残課題**:
- 日報の「課題・問題点」から未解決のもの
- 日報の「作業内容」で完了していないもの

### 直近の流れ（過去 2-3 日）

複数日の日報がある場合、ここ数日の作業の流れを 3-5 行で要約する。
1 日分しかなければこのセクションは省略する。

### 今日やること

- 日報の「翌日の予定」セクションから転記
- 残課題の中で今日着手すべきもの
- 優先度順に並べる

### 注意事項

- 未解決のブロッカーや依存関係があればここに記載
- チームへの確認・共有が必要な事項
```

## 注意事項

- 出力の前後に余計な説明を付けない。ブリーフィングのみを出力する。
- 日報がない日は飛ばす。「日報がありません」とは報告しない。
- 日報の情報量に応じて各セクションの粒度を調整する。少なければ簡潔に。
- 「今日やること」が空の場合（翌日予定が書かれていない場合）は、残課題から推定して提案する。
